Project: Timecard Automation for Supplier Invoicing

In large-scale procurement environments, suppliers submit timecards under Time & Material (T&M) or Fixed Price models. Previously, this was manual—causing invoice delays, compliance issues, and inefficiencies in Oracle SaaS receipt processing.

We built a VBCS-based app for 50+ suppliers to submit timecards, integrated with ATP and Oracle Cloud via OIC. Key features:

  • T&M and Fixed Price modes
  • PO/BPA validation
  • Automated approval workflows
  • BI Publisher reports for compliance
  • Dynamic UI with validations and fragments in VBCS
  • Normalized ATP schema and REST-exposed APIs via ORDS
  • OIC integrations for PO sync and receipt creation
  • Email notifications with retry-safe queues
  • BI Publisher reports for PDF/Excel downloads
  • 60% reduction in manual validations
  • 80% faster processing time
  • 10,000+ timecards processed annually
  • 100% contract compliance and audit success

Project: Job and Program Management Applications

The client, a multi-entity global organization, required a centralized platform to manage job numbers and their associated employee and department assignments. The absence of a unified system led to data fragmentation, manual dependency on Excel for mapping, and significant delays in workforce planning and reporting. They also lacked audit visibility for changes made to assignments over time.

I designed and built two tightly integrated Oracle VBCS applications: one for Job Management and one for Program Management. These applications provided real-time tracking, editing, and reporting of job and assignment data through seamless interaction with Oracle ATP and REST services via OIC.

  • Support for one-to-many relationships (job-to-employee, job-to-department)
  • Modular form rendering using dynamic `vb-fragments`
  • History tracking and audit trail on all changes
  • Bulk insert and update with validation via PL/SQL packages
  • Data integration with HCM for employee/department fetch
  • Designed normalized database schema in ATP for job details and assignments
  • Created reusable PL/SQL packages for validation, bulk insert, update, and rollback
  • Built REST-exposed procedures using ORDS and secured access using role-based control
  • Developed dynamic VBCS forms using `oj-for-each`, `oj-collapsible`, and `vb-fragments`
  • Implemented audit logic with timestamped change tracking and user metadata
  • Created helper UI fragments for people search, auto-fill from HCM, and assignment review
  • Enabled centralized control over 5,000+ job records and 15,000+ assignments
  • Reduced manual reconciliation time by over 60%
  • Improved traceability with 80% enhancement in audit visibility
  • Cut onboarding and job role assignment effort from 3 days to 1
ChallengeResolution
Rendering dynamic forms based on assignment count Used `oj-for-each` and collapsibles to create fully dynamic UI blocks
Validating foreign key links in nested assignments Used PL/SQL validation packages and error maps for real-time feedback
Handling bulk operations with rollback logic Built transaction-safe PL/SQL packages with EXCEPTION blocks and audit
  • Nested dynamic forms in VBCS are manageable using component modularity
  • Audit logging is critical in high-change applications and should be schema-level
  • HCM integrations require throttling and optimization for REST fetches
  • Bulk PL/SQL procedures must be built with idempotent design principles
  • Excel/CSV upload support with preview-validation before insert
  • Role-based dashboards for department heads to approve assignments
  • Job assignment visualization using Oracle JET charts
  • Drill-down exportable history view for audit and compliance team

Project: Oracle Fusion Automation Suite

The client was undertaking a large-scale Oracle Cloud Fusion implementation across Financials (GL), Project Portfolio Management (PPM), and Supply Chain Management (SCM). Manual testing of business processes was time-consuming and error-prone, often resulting in delayed releases, inconsistent test coverage, and higher defect leakage during UAT. A full automation suite was needed to support regression testing, data validation, and migration efforts.

I led the development and delivery of an end-to-end automation framework using UFT One and ALM to cover over 1,500 test scenarios. Additionally, I designed a custom Angular-based UI that enabled business analysts and QA leads to perform data transformations and validations for pre-UAT cycles. The suite covered:

  • 1500+ automated test scripts across GL, PPM, and SCM modules
  • Daily scheduled regression runs with ALM integration
  • Reusable components for login, navigation, and transaction validation
  • BI Publisher reports for consolidated test results
  • Angular UI tool for test data preparation and migration validation
  • Defined test automation strategy with Deloitte QA leadership
  • Built modular reusable functions for rapid script development
  • Created end-to-end workflows for GL Journal Entries, Project Costing, and PO Lifecycle
  • Integrated automated scripts with ALM for defect reporting and test cycle tracking
  • Developed a custom UI using Angular to allow non-technical users to validate data mappings and initiate test cycles
  • Generated BI Publisher reports from ALM and UFT logs for executive reporting
  • 70% reduction in manual testing effort across releases
  • 50% faster data transformation and migration validation
  • Enabled parallel testing across 3 major Fusion tracks (GL, PPM, SCM)
  • Reduced UAT cycle from 4 weeks to 2.5 weeks
  • Improved defect detection rates during SIT by over 40%
